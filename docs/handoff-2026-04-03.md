## Bugs & Issues Discovered

- **CI build order matters**: `npm run build --workspaces` can run in parallel, causing failures when core/installer need shared built first. Fixed by building sequentially in CI.
- **ESM-only deps**: chalk 5+, ora 6+, boxen 7+ are ESM-only. Installer uses CJS-compatible versions (chalk 4, ora 5). Future migration to ESM may be needed.
- **Deploy workflow fails after rename**: First auto-deploy after merge failed because VM path was `~/obsidian_open_brain` but deploy.yml referenced `~/lox-brain`. Fixed with symlink.

## Key Decisions & Gotchas

- **Name "Lox"** from Latin *locus*. Repo: `lox-brain`. CLI/product: just "Lox".
- **Monorepo with npm workspaces** (not pnpm/yarn). `workspace:*` protocol does NOT work with npm — use `"*"` for workspace deps.
- **`tsconfig.base.json` with `composite: true`** — each package has its own tsconfig extending base. Don't commit `.tsbuildinfo` files.
- **MCP config uses `source` not `xargs`** — `bash -c 'set -a && source /etc/lox/secrets.env && set +a && node ...'` avoids issues with comments in env files.
- **Secrets in `/etc/lox/secrets.env`** (not `.env` in repo) — Zero Trust, chmod 640.
- **Two vault presets**: Zettelkasten (Eduardo's model, recommended) and PARA (Tiago Forte). Both offered during install.
- **Multi-user prep**: `vpn_peers[]` array in config, `created_by TEXT` column in schema, `--mode=personal|team` flag.
